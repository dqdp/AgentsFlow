# Domain Pack: coding-agent

## Purpose

Rules for coding-agent workflows such as Codex/Claude/Gemini-style implementation agents.

## Rules

- Do not edit unrelated files.
- Do not weaken tests to make verification pass.
- State commands run and results.
- Ground plans in repository evidence.
- Preserve accepted ADRs unless user approves changes.


## Usage

Workflows may load this pack to parameterize contract writing, scenario design, impact mapping, and review.
