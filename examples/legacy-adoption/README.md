# Legacy Adoption Example

This example shows how an existing project with old agent/process artifacts can adopt AgentsFlow without creating ambiguous instruction authority.

## Scenario

The project already has:

```text
AGENTS.md
old-skills/
Docs/implementation-history/
```

The selected mode is `knowledge-extraction`: useful knowledge is extracted into AgentsFlow project packs and workflow bindings, while original legacy artifacts are archived or marked non-authoritative.

## Artifacts

Use the templates:

```text
templates/legacy-agent-system-inventory.json
templates/legacy-adoption-decision.yaml
templates/agent-instruction-conflicts.md
templates/agent-instruction-migration-plan.md
templates/legacy-backup-manifest.yaml
templates/active-instruction-map.yaml
```
